## Pipeline Stages

| Stage | Description |
|-------|-------------|
| Checkout | Pulls latest code from GitHub |
| Install Dependencies | Runs `npm install` in backend |
| Build | Runs `npm run build` |
| Test | Runs Jest unit tests, publishes JUnit report |
| Deploy | Builds Docker images and pushes to Docker Hub |

## Pipeline Configuration

### How I Configured the Pipeline

1. Installed Jenkins on localhost:8080 using the Windows installer from jenkins.io
2. Installed required plugins: NodeJS Plugin, Pipeline, GitHub Integration, Docker Pipeline
3. Configured Node.js LTS v20 in Manage Jenkins → Tools → NodeJS
4. Added GitHub Personal Access Token as credentials in Jenkins
5. Added Docker Hub credentials in Jenkins (ID: `docker-hub-creds`)
6. Created a new Pipeline job pointing to this GitHub repository
7. Set Script Path to `Jenkinsfile`
8. The pipeline auto-triggers on every push to the `main` branch

### Test Configuration

Tests use Jest with Supertest to test all API endpoints without needing a real database. The PostgreSQL pool is mocked using `jest.mock('pg')` so tests run completely offline.

JUnit XML reports are generated by `jest-junit` and published in Jenkins under **Test Results**.

## Challenges Faced

1. **Database connection in tests** — Tests would fail trying to connect to PostgreSQL. Fixed by mocking the `pg` module with `jest.mock('pg')` so no real DB connection is needed during testing.

2. **Jest not found** — Had to install Jest as a dev dependency and configure the test script in `package.json` explicitly.

3. **JUnit report path** — Jenkins couldn't find the JUnit XML file until the correct relative path `todo-app/backend/junit.xml` was set in the Jenkinsfile.
